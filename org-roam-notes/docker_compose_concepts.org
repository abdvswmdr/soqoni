#+TITLE: Docker Compose Concepts
#+ROAM_TAGS: docker docker-compose orchestration

[[file:docker_compose_commands.org][Docker Compose Commands]] are used to manage the lifecycle of multi-container Docker applications.

The =setup/docker-compose.yml= file defines a multi-service application. Docker Compose is a tool for defining and running multi-container Docker applications. It uses a YAML file to configure the application's services.

* Key Concepts in =docker-compose.yml=

** =version=
   - Specifies the version of the Docker Compose file format. Version '3.0' is used here.

** =services=
   - This is where you define the different containers that make up your application. Each service corresponds to one container.
   - In =setup/docker-compose.yml=, the services are: =control=, =frontend=, =catalogue=, =carts=, =carts-db=, =jenkins=, and =docker=.

** =image=
   - Specifies the Docker image to use for the container. This can be an image from Docker Hub or a custom-built image.
   - Example: =image: mongo=

** =build=
   - Used to build a custom image for the service.
     - =context: ./= specifies the build context (the directory containing the Dockerfile).
     - =dockerfile: Dockerfile= specifies the name of the Dockerfile.
   - The =jenkins= service has a =build= section, meaning it will be built from the =setup/Dockerfile=.

** =ports=
   - Maps ports between the host machine and the container.
   - Format: ="HOST_PORT:CONTAINER_PORT"=
   - Example: ="8080:8080"= maps port 8080 on the host to port 8080 in the =jenkins= container.

** =networks=
   - Defines the networks that the services will connect to.
   - A custom bridge network =custom= is created with a specific subnet =192.168.61.0/24=. This allows containers to communicate with each other using predictable IP addresses.
   - Each service is assigned a static IPv4 address within this network.

** =volumes=
   - Used for persisting data generated by and used by Docker containers.
   - Named volumes like =jenkins-data= are managed by Docker.
   - Bind mounts like =- jenkins-data:/var/jenkins_home= mount a host path or named volume into the container. In this case, it persists the Jenkins home directory.

** =environment=
   - Sets environment variables inside the container.
   - The =jenkins= container uses this to configure the Docker host it should connect to (=DOCKER_HOST=).

** =depends_on=
   - Defines dependencies between services.
   - =jenkins= =depends_on: - docker= means that the =docker= service will be started before the =jenkins= service.

** =privileged=
   - The =docker= service runs in =privileged= mode. This is required for Docker-in-Docker (dind) to function correctly, as it needs extended kernel privileges.
