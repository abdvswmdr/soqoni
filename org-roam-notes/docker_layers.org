#+TITLE: Docker Images, Containers, and Layers
#+file_tags: :docker:image:layer:container:

Docker images and containers are built upon the concept of =layers=. This is a fundamental part of what makes Docker efficient and fast.

* Docker Images and Layers

A Docker image is a read-only template used to create containers. This template is not a single, monolithic file; instead, it's composed of a series of stacked, read-only layers.

- *Layered Filesystem*: Think of it like a stack of transparent sheets. Each sheet has something written on it, and when you look down through the stack, you see the combined result.
- *Dockerfile to Layers*: Each instruction in a =Dockerfile= (like ~FROM~, ~RUN~, ~COPY~) creates a new layer in the image.
- *Caching and Efficiency*: This layered architecture is key to Docker's efficiency. When you rebuild an image, Docker only rebuilds the layers that have changed. If you change a command at the end of your Dockerfile, Docker reuses all the unchanged layers from its cache, making builds much faster.

* Docker Containers and the Writable Layer

When you create a container from an image, Docker does not copy the entire image. Instead, it adds a thin =writable layer= on top of the stack of read-only image layers.

- *Container Layer*: This top layer is where all changes made inside the running container are stored. If you modify a file, create a new file, or delete a file, it happens in this writable layer.
- *Copy-on-Write (CoW)*: The underlying image layers are ~never~ modified. If a container needs to modify a file that exists in a lower, read-only layer, Docker copies the file up to the writable layer first, and then the container modifies the copy. The original file in the image layer remains untouched. This is called a =copy-on-write= strategy.

This means multiple containers can share access to the same underlying image layers without interfering with each other.

* Inspecting Image Layers

You can inspect the layers of an image using the command line.

** =docker history=
   - This command shows the history of how an image was created, with each line corresponding to a layer. It shows the command that created the layer and the size of the layer.

#+begin_src shell
# Usage: docker history <image_name>
docker history jenkins/jenkins:2.480-jdk17
#+end_src

** =docker inspect=
   - This provides detailed low-level information about Docker objects, including the layers of an image in JSON format. The layers are listed under ~RootFS.Layers~.

#+begin_src shell
# Usage: docker inspect <image_name>
docker inspect jenkins/jenkins:2.480-jdk17
#+end_src
