* Soqoni — Infrastructure

Deployment and CI/CD infrastructure for the Soqoni microservices e-commerce platform. Provisions four services across Docker Compose nodes using Ansible, automated with a Jenkins pipeline, and fully deployable to Kubernetes with Istio service mesh and Prometheus monitoring.

** Features

- One-command stack startup with Docker Compose (~sudo docker compose up -d~)
- Jenkins CI/CD pipelines for automated build, test, and Docker Hub push
- Ansible roles deploy each service idempotently to isolated nodes
- Kubernetes manifests with Ingress, HPA, and rolling update strategy
- Istio service mesh — mTLS, traffic splitting, canary deployments, fault injection
- Prometheus + Grafana monitoring via ~kube-prometheus-stack~
- Kiali topology graph and Jaeger distributed tracing

** Kubernetes Deployment

#+BEGIN_SRC bash
# Prerequisites: minikube, kubectl, helm

minikube start --cpus=4 --memory=8192
minikube addons enable ingress

# Apply all manifests
kubectl apply -f ../soqoni-k8s/

# Install monitoring stack
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm install monitoring prometheus-community/kube-prometheus-stack

# Install Istio
istioctl install --set profile=demo -y
kubectl label namespace default istio-injection=enabled

open http://$(minikube ip)
#+END_SRC
